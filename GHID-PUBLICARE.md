# Ghid de publicare permanentă — GitHub Pages

De ce această cale: găzduire gratuită permanent, HTTPS automat, fișiere până la 100 MB, iar controlul site-ului stă într-un singur cont — al vostru. Se face o singură dată, în vreo 10 minute.

## Partea A — pașii tăi, o singură dată

1. **Contul**: github.com → Sign up → e-mail, parolă puternică și unică (păstrată într-un loc sigur), nume de utilizator (ex: `atelier-gabriela`). Nu cere card.
2. **Securizarea** (cele mai importante 5 minute): click pe poza de profil, dreapta-sus → Settings → Password and authentication → Two-factor authentication → Enable → alege metoda cu aplicație de autentificare → salvează codurile de recuperare undeva offline (tipărite sau scrise).
3. **Depozitul**: butonul „+" din dreapta-sus → New repository → Repository name: `atelier-gabriela` → vizibilitate **Public** → Create repository.
4. **Încărcarea**: dezarhivează pachetul primit de la mine → în pagina depozitului: Add file → **Upload files** → trage TOT conținutul dezarhivat (index.html, stil.css, date.js, mecanica.js, folderul `fonturi`, 404.html, robots.txt, README.md și folderul `poze` întreg — browserul urcă folderul cu tot cu conținut) → jos apasă **Commit changes**.
5. **Activarea Pages**: în tab-ul Settings al depozitului → Pages (meniul din stânga) → la Build and deployment: Source = **Deploy from a branch**, Branch = **main**, folder = **/(root)** → Save.
6. **Verificarea**: în 1–2 minute, sus pe aceeași pagină apare adresa `https://NUMELE-TAU.github.io/atelier-gabriela/`. Deschide-o pe telefon — totul trebuie să funcționeze, cu galerie și animații. Când opțiunea devine activă, bifează **Enforce HTTPS**.
7. **Trimite-mi adresa** în conversație.

## Partea B — ce fac eu imediat după ce primesc adresa

Setez adresa completă la imaginea de previzualizare (ca Facebook și WhatsApp să afișeze coperta când e distribuit linkul), generez sitemap-ul pentru Google și populez site-ul pe măsură ce sosește conținutul — îți trimit de fiecare dată fișierele noi, gata de reîncărcat.

## Partea C — cum se actualizează site-ul de acum înainte

Orice fișier nou de la mine: pagina depozitului → Add file → Upload files → tragi fișierele (cele cu același nume se suprascriu automat) → Commit changes. Atenție: interfața web acceptă maximum ~100 de fișiere per încărcare — pentru ziua arhivei (sute de poze), urmează coregrafia din ZIUA-ARHIVEI.md: GitHub Desktop dintr-un foc, sau web în tranșe. Site-ul se actualizează singur în circa un minut, iar istoricul versiunilor rămâne — orice pas greșit se poate întoarce.

## Note de siguranță

Depozitul e public și e normal să fie: site-ul însuși e public, iar pozele trec prin curățarea mea de metadate ascunse înainte de publicare. Nu urcați aici nimic privat — arhiva de Facebook are depozitul ei separat și temporar, conform planului. Coperta actuală din `poze/coperta.jpg` e desenată de mine ca substitut elegant; când avem fotografiile reale, o înlocuim cu o poză orizontală frumoasă a unui produs. Singura cheie a întregului site rămâne contul: parola și verificarea în doi pași sunt tot ce trebuie păzit.

## Opțional, mai târziu: domeniu propriu

Când veți dori o adresă proprie (ex: `atelierulgabrielei.ro`), se cumpără domeniul de la un registrar românesc (~50 lei/an) și se leagă din Settings → Pages → Custom domain, în cinci minute. Nu e nevoie acum — adresa github.io e permanentă și gratuită.
